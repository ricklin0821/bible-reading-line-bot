# 📖 聖經讀經計畫 LINE Bot

一個功能完整的 LINE Bot，提供一年期聖經讀經計畫、每日進度追蹤、填空測驗和每日推送通知功能。使用中文和合本聖經經文，支援兩種讀經計畫（正典順序和均衡計畫）。

## 🚀 中期功能規劃 (Roadmap)

以下是我們正在規劃開發的中期功能，旨在提升社群互動與個人成就感：

### 1. **小組功能 (Group Feature)**
- **目標**: 透過小組互動，增強成員間的連結與讀經動力。
- **核心機制**:
    - **隨機分組**: 系統自動將使用者隨機分配至最多 6 人的小組。
    - **進度通知**: 當小組內有成員完成當日讀經進度時，系統會自動發送通知給同組的其他成員，以達到互相激勵的效果。
    - **小組留言板**: 組員可以在專屬的小組空間中留言、分享心得、彼此代禱。
    - **使用者自主權**: 
        - **隨機換組**: 如果使用者想更換小組環境，可以隨時選擇「隨機換組」。
        - **取消通知**: 使用者可以自行選擇是否接收小組的進度通知。

### 2. **成就分享 (Achievement Sharing)**
- **目標**: 提供使用者一個方式來分享他們的讀經成就，擴大正向影響力。
- **核心機制**:
    - **生成精美成就圖**: 當使用者達成特定里程碑時（例如：連續讀經 30 天、完成特定書卷、達到積分門檻等），系統會自動生成一張設計精美的成就圖片。
    - **客製化內容**: 圖片上會包含使用者的姓名（或暱稱）、達成的成就、以及一句鼓勵的經文。
    - **一鍵分享**: 提供方便的按鈕，讓使用者可以輕鬆將成就圖片分享到各大社群媒體，如 Instagram, Threads, Facebook, LINE 等。

## 🏗️ 技術架構

- **後端框架**: FastAPI (Python)

- **部署平台**: Google Cloud Run
- **圖片生成**: Pillow
- **版本控制**: Git & GitHub

## 🚀 快速開始

### 前置需求

1. **LINE Developers 帳號**: 用於建立 Messaging API Channel。
2. **Google Cloud 帳號**: 用於部署 Cloud Run 與使用 Firestore。

### 部署流程

```bash
# 1. Clone 專案
git clone https://github.com/ricklin0821/bible-reading-line-bot.git
cd bible-reading-line-bot

# 2. 設定環境變數 (請參考 .env.example)

# 3. 部署到 Google Cloud Run
gcloud run deploy bible-bot --source . --region asia-east1 --project YOUR_PROJECT_ID
```

詳細的部署步驟與設定，請參考專案中的相關文件。
